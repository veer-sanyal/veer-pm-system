**Product Management / TPM**
Order: Education → Experience (Firmly first, SMIF second) → Projects (StudyFlowForge) → Leadership (EPICS) → Skills.
Lead Firmly bullets with requirements/roadmap/metrics framing. Highlight IBE as rare biz+eng hybrid in cover letter. Use "Product & Strategy" as first skills category.

**Software Engineering / Full-stack / Product Engineering**
Order: Education → Projects (StudyFlowForge first) → Experience (Firmly) → Leadership (EPICS) → Skills. SMIF goes to Leadership or omitted.
Lead StudyFlowForge bullets with full-stack scope, SQL schemas, multi-factor scoring model. Lead Firmly with Node.js/Express + QA workflow. Use "Full-Stack & Product" or "Languages & Data" categories.

**Finance / IB / Equity Research / S&T**
Order: Education → Experience (SMIF first, Firmly second) → Projects (StudyFlowForge as quant/scoring story) → Leadership (EPICS) → Skills.
Lead with $400K AUM + U.S. Bancorp report + DCF. Emphasize Excel + financial modeling. Reframe StudyFlowForge as "multi-factor scoring model" / "operational dashboards" / quantitative work. Use "Microsoft Office" or "Analytics & Reporting" first.

**Consulting**
Similar to finance ordering but emphasize stakeholder presentation, structured analysis, tradeoff framing. EPICS gets more weight (team leadership, structured recommendations to stakeholders).

**AI / ML / Data**
Lead StudyFlowForge with Gemini API + spaced repetition model + dashboards. Mention Claude Code usage. Use "AI & Data" as first category.

**AI data-rater / evaluation roles in a domain (e.g., Handshake AI "Finance Expert")**
Pay rate is set by the domain (finance), so position as a domain expert FIRST (lead SMIF, finance coursework, financial modeling/statements), then use Veer's real AI-output-review experience as the differentiator: Firmly QA = reviewing/validating output for accuracy across edge cases; StudyFlowForge = integrated + evaluated Gemini outputs before surfacing to users. Mirror listing terms: "review AI-generated responses for accuracy/analytical rigor/clarity," "independent + asynchronous," "attention to detail," "following written guidelines." Reframe Firmly bullets toward review/validation/judgment rather than building. Done for Handshake AI Finance Expert (Bachelor's), May 2026.

**AI creative-specialist roles in a domain (e.g., Handshake AI "2D & 3D Digital Media", $125/hr)**
Pay is set by the creative domain, so lead as a designer FIRST. Order: 1-line summary headline → Design Experience (Etsy) → Design Projects (portfolio + StudyFlowForge) → Additional Experience (Firmly, review/QA framing) → Education → Leadership → Skills. The listing names open-source tools (GIMP, Inkscape, Krita, Libresprite, Blender); mirror those exact names in the Skills "Creative Tools" line and frame Adobe work as transferring directly to them (honest — Veer is Adobe-native, thin on Blender/3D, and the "OR" wording means 2D proficiency alone qualifies). Differentiator = real AI-output evaluation (StudyFlowForge Gemini review; Firmly guideline-validation) reframed as "creating and rating data used to train AI." A short headline earns its space here because the role values creativity and the human skim must see designer-fit in seconds. Done for Handshake AI 2D/3D Digital Media, 2026-06-21.

**Fintech (Stripe, Brex, etc.)**
Hybrid: lead Firmly (PayPal checkout = payments domain) then StudyFlowForge. Use "Full-Stack & Product" category.

TypeScript/JavaScript/React/Node/SQL/APIs are framed as "built with" (AI-assisted) or described through the project bullet, never as standalone proficiencies. See [[user-skills-inventory]] calibration block. Project bullets describing real artifacts are fine; flat skill claims are the inflation risk.
- Resume is 1 page hard cap.
- Always include EPICS for leadership signal (1–2 bullets).
- Match skills category names to job description framing.
- Top third of page must contain strongest proof for the specific role.
